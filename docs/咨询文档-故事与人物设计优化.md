# 故事与人物设计优化咨询文档

## 一、项目背景

### 1.1 项目概述

**项目名称**：《What's the time》

**项目类型**：多周目文字冒险游戏（Visual Novel）

**技术栈**：Bun + TypeScript + Vue 3

**开发阶段**：剧本设计与系统开发并行进行中

### 1.2 游戏核心机制

- **时间输入机制**：玩家在每个周目开头输入时间（HH:MM），游戏据此播放对应的记忆片段
- **多周目设计**：单个周目流程短（约10-15分钟），通过多周目逐步揭示真相
- **两问一锁机制**：关键时间点需要两次输入（主时间+辅助时间）来区分不同事件
- **Flag系统**：通过解锁条件控制剧情流程，多周目间信息可累积

### 1.3 叙事风格与目标

- **叙事风格**：受 David Lynch 影响，非线性叙事，大量使用叙事诡计
- **表层体验**：伪装成轻松的校园恋爱喜剧
- **内核真相**：预谋杀人、记忆扭曲、叙事操控
- **Meta元素**：在关键节点打破第四面墙，将玩家拉入责任链

## 二、故事核心设定

### 2.1 故事主线

**核心真相**（玩家在游戏中逐步发现）：

1. **两起杀人案**：
   - **一年前（A案）**：主角在校园楼顶于 **10:07** 有计划地杀死恋人 A（名字：X）
   - **现在（B案）**：主角将班上同名女生 B（名字：X）视作替代品，在类似心理驱动下再次杀死她，同样围绕 10:07

2. **栽赃与自我受害叙事**：
   - A案中，主角精心布局，将责任栽赃给替罪羊 H
   - 主角持续塑造"我只是受害者"的形象，逐渐也相信了自己的谎言

3. **代偿人格与"她"的模糊身份**：
   - 杀死A后，主角在内心形成理想化的"她"（不会背叛、永远理解他）
   - 遇到同名B后，主角将理想化人格强行投射在B身上，忽视B的真实性格
   - 游戏中"她"长期不区分是A还是B，直到真相周目才拆分

4. **时间与记忆的扭曲**：
   - 主角被第一次杀人的 **10:07** 困住
   - 多周目是他反复重播、剪辑两起案件相关记忆的过程
   - 玩家输入时间，实际是在操纵"我"愿意面对的记忆片段

5. **最终meta突破**：
   - 在最后一个关键周目，"她"打破第四面墙，要求玩家输入现实中的"现在时间"
   - 将"叙事时间/游戏时间/现实时间"折叠，把玩家也拖入"你在重播这场暴力"的责任中

### 2.2 叙事诡计设计

1. **代词模糊**：大量使用"他/她/那个人/那件事"，延后真名出现
2. **时间不标日期**：只给出"几点几分"，让不同年份的同一时间在玩家认知中重叠
3. **同名混淆**：A与B同名（都是X），E想成为新的"她"，构成"谁配得上这个名字"的暗线
4. **角色自知/玩家不知**：主角隐藏大量真相，玩家需要通过外部信息拼凑真相
5. **虚假选择**：在回忆B案时，提供看似有意义的选项，让玩家误以为自己做出了"杀死B"的决定

## 三、主要人物设定

### 3.1 "我"（主角/Protagonist）

**表层印象**：
- 性格内向、略显木讷的学生，成绩中上
- 在旁人眼中，是"一年前那件事中最可怜的人"，被同情、被特殊照顾
- 所有怪异行为被解释为创伤后遗症

**真实面貌**：
- **极端的自私与控制欲**：内心深处只有自己，强烈的控制欲，无法容忍任何形式的"失去控制"
- **虚伪与表演**：面具不是保护壳，而是武器。完全清楚自己在做什么，但选择用"创伤后遗症"来掩盖
- **两起案件的实施者**：对A是有计划的预谋杀人+栽赃；对B是模式化重复犯罪+分尸抛尸
- **推责机制**：不仅对他人推责（如栽赃给H），也会对玩家推责，通过嘲讽将责任转嫁给玩家

**核心冲突**：
- 一边坚持自己是"被事件选中的受害者"，一边不断复刻加害行为
- 被时间（10:07）和同名"她"困住，拒绝前进

### 3.2 A（旧案受害者，名字：X）

**性格特质**：
- 独立、有主见，对控制行为敏感
- 重视关系的平等和尊重
- 逐渐意识到与主角的关系不健康，决定离开

**关键事件**：
- 在H的提醒下，意识到问题的严重性，决定必须离开
- 使用纸条约主角去天台"好好谈谈"，想给关系一个"正式结束"
- 主角利用这个机会，精心设计陷阱，让H成为"凶手"，A在10:07坠落

**叙事功能**：
- 作为"理想她"的原型，被主角在内心美化和扭曲
- 名字与B完全相同，都是X，这是叙事诡计的核心

### 3.3 B（现案受害者，名字：X）

**表层印象**：
- 与A同名（X），但性格明显不同（更随性/更阳光/更直率）
- 新学校的同学，起初真心把主角当"需要照顾的可怜人"

**真实内心**：
- 有明确的人生目标，对被他人"束缚"非常敏感
- 逐渐意识到主角看她的眼神里有"别人"的影子
- 约主角"好好谈谈"，想明确拒绝/划清界限："我不是她"

**弧光**：
- 前期：轻松日常线女主，玩家容易把她当作"攻略对象"
- 中期：开始在对话中试探："你是不是把我当成谁了？"
- 后期：选择约主角"好好谈谈"，这次谈话成为第二次杀人的直接导火索

### 3.4 H（旧案中的替罪羊）

**真实情况**：
- 完全无辜，整个事件都是主角故意制造的陷阱
- 对A有好感，但从未明确表达
- 察觉到A和主角关系的问题，主动告诉A这段关系不健康
- 主角利用H的善意，让H以为A在天台有危险，需要帮助
- H赶到天台想救A，但在救的过程中，由于主角提前做的手脚，A意外坠落
- H的动机是保护A，但主角利用了这个善意，让H成为"凶手"

**叙事功能**：
- 揭示主角如何故意制造陷阱，以及如何通过栽赃来逃避责任
- 展现"偏见"如何成为加害者栽赃的工具

### 3.5 E（一年前的目击者+现案共犯）

**身份**：
- 一年前与主角在同一所高中的同学，因为喜欢主角也选择了同一所新学校
- 成为连接前后两件事的证人

**一年前的秘密**：
- 因为喜欢主角，偷偷跟踪他，看到主角在天台做了手脚
- 没有看到推搡过程，只看到了A坠落的结果
- 选择沉默，事后长期用"可能是错觉"说服自己，形成强烈负罪感

**现在的选择（B案，共犯）**：
- 在B案当天，E清楚地看到主角杀人的过程
- 她没有报警，而是在晚上18:40左右，出现在主角准备分尸/处理尸体的现场
- 做出极端选择：对主角说"叫她C""她会帮他处理掉C"
- 这里的C是高度压缩的符号：既指旧案中的A，也指现在B的尸体，更指围绕这个名字的一切罪和纠缠

**与主角的情感关系**：
- 她的爱里有真实的心动，也有逃避
- 不愿承认自己喜欢的是"杀人犯"，所以选择加入犯罪，把自己也变成"同一种人"
- 她想成为主角生命中独一无二的"共犯"和"支撑点"，所以主动抢占"C这个位置"

### 3.6 D（B的朋友/同学）

**身份**：
- B的好友/室友/社团同伴，对B的日常状态最敏感的人
- "最后一次见到B和主角在一起"的见证人

**情感与立场**：
- 核心驱动力是"为B说话"，而不是"破案"
- 大家都说"她可能只是有事回家"，只有D坚定认为"她不会不打招呼就消失"

**弧光**：
- 前期：调侃主角和B像情侣，帮他们打圆场
- 中期：逐渐收集矛盾信息，对主角态度从中立→怀疑
- 后期：明确质问主角："那天你到底对她说了什么？"

### 3.7 其他角色

- **F（班主任/导师）**：代表"制度/成年人"的视角，对主角的态度从同情→不安→摇摆
- **G（警官）**：一年前旧案、现在失踪案都参与过的警官，在问话中使用大量模糊代词，让旧案与新案在语言层面缝合
- **C（概念性的"位置"）**：不再是某个具体人的名字，而是一个"角色位"——那个被社会故事认定为"那件事中的怪物/关键人物"的位置

## 四、需要咨询的核心问题

### 4.1 人物塑造与动机

#### 问题1：主角"我"的复杂性
- **现状**：主角被设计为一个完全清醒的加害者，通过精心设计的表演来逃避责任。他利用"受害者"身份操控他人，推责给玩家。
- **疑问**：
  1. 这种"完全清醒的恶"是否会让角色显得过于扁平？是否需要一些更复杂的心理层次？
  2. 主角的"家庭悲剧"背景（控制、暴力、情感虐待）如何更好地服务于角色塑造？是否需要在游戏中更明确地展现？
  3. 主角对玩家的推责机制，如何在保持冲击力的同时，避免让玩家感到被冒犯或过于不适？

#### 问题2：A与B的区分度
- **现状**：A和B同名（都是X），这是叙事诡计的核心。A更独立、有主见，B更随性、阳光。但她们都因为拒绝成为主角的"理想她"而被杀。
- **疑问**：
  1. 如何在保持同名混淆的同时，让A和B在性格和命运上有足够的区分度，避免角色同质化？
  2. A的"觉醒"和B的"觉醒"是否有足够的差异？如何让两次"觉醒"在叙事上形成递进而非重复？
  3. 如何让玩家在真相周目时，能够清晰地感受到"这是两个完全不同的人，却被同一个名字绑定"的悲剧感？

#### 问题3：E的共犯动机
- **现状**：E因为喜欢主角，从一年前的沉默目击者，变成B案的主动共犯。她想成为主角生命中独一无二的"共犯"和"支撑点"。
- **疑问**：
  1. E的动机（"不愿承认自己喜欢的是杀人犯，所以选择加入犯罪"）是否足够有说服力？是否需要更复杂的心理层次？
  2. E的"自我牺牲"逻辑如何更好地展现？她的扭曲是否需要有更明确的转折点？
  3. E与主角的关系，如何在保持"扭曲的爱"的同时，避免让E显得过于工具化？

#### 问题4：H的替罪羊设定
- **现状**：H完全无辜，被主角精心设计的陷阱陷害。他的动机是保护A，但主角利用了这个善意。
- **疑问**：
  1. H的"坏名声"（暴躁、打架、逃课）如何更好地服务于"偏见成为栽赃工具"的主题？是否需要更细致的展现？
  2. H在游戏中的呈现（多通过他人口中出现，很少正面出场）是否足够？是否需要给他更多的存在感？
  3. 如何让玩家在中期相信"也许真的是H干的"，同时在后期反转时不会感到被欺骗？

### 4.2 故事结构与节奏

#### 问题5：叙事诡计的平衡
- **现状**：游戏大量使用叙事诡计（代词模糊、时间模糊、同名混淆、虚假选择等）。
- **疑问**：
  1. 如何在保持神秘感的同时，避免让玩家感到困惑或挫败？
  2. 叙事诡计的"揭示"时机如何把握？何时应该开始给玩家线索，何时应该保持模糊？
  3. 多周目设计中，如何确保玩家有足够的动力继续探索，而不是因为信息不足而放弃？

#### 问题6：时间线的清晰度
- **现状**：游戏刻意模糊时间，只给出"几点几分"，让不同年份的同一时间在玩家认知中重叠。
- **疑问**：
  1. 如何在保持时间模糊的同时，让玩家能够逐步拼凑出真实的时间线？
  2. 两问一锁机制（主时间+辅助时间）是否足够清晰？玩家是否能够理解这个机制？
  3. 如何避免玩家因为时间混乱而感到挫败，同时保持"需要推理"的挑战性？

#### 问题7：Meta元素的冲击力
- **现状**：在最后一个关键周目，"她"打破第四面墙，要求玩家输入现实中的"现在时间"，将玩家拉入责任链。
- **疑问**：
  1. Meta突破的时机是否合适？是否需要在更早的周目就埋下伏笔？
  2. 如何让Meta元素在保持冲击力的同时，不会显得突兀或过于说教？
  3. 玩家对Meta元素的反应如何预测？如何确保玩家能够理解并接受这个设计？

### 4.3 主题表达与深度

#### 问题8：主题的清晰度
- **现状**：游戏试图探讨多个主题：控制欲、推责机制、叙事操控、沉默的共犯、偏见与栽赃等。
- **疑问**：
  1. 这些主题如何更好地整合？是否有核心主题需要更突出？
  2. 如何在保持主题深度的同时，避免让游戏显得过于说教或沉重？
  3. 游戏的"教育意义"和"娱乐性"如何平衡？

#### 问题9：暴力与死亡的呈现
- **现状**：游戏涉及两起杀人案，B案还涉及分尸等暴力内容。
- **疑问**：
  1. 如何在保持冲击力的同时，避免过度渲染暴力？
  2. 暴力的呈现方式（文字描述、暗示、留白）如何选择？
  3. 如何确保游戏不会因为暴力内容而失去部分受众？

#### 问题10：玩家体验与责任
- **现状**：游戏试图让玩家体验"被推责"的感受，最终意识到"重复观看和操作暴力本身也是一种参与"。
- **疑问**：
  1. 如何让玩家在体验"被推责"时，不会感到被冒犯或过于不适？
  2. 玩家的"责任"如何界定？游戏是否应该给玩家"选择"的机会？
  3. 如何确保玩家能够从游戏中获得反思，而不是单纯的负面情绪？

## 五、具体咨询方向

### 5.1 人物塑造优化

**期望反馈方向**：
1. 各角色的动机是否足够有说服力？是否需要调整或补充？
2. 角色之间的关系是否足够复杂和有趣？是否有遗漏的重要关系？
3. 角色的弧光是否清晰？是否有足够的成长或变化？
4. 是否有角色显得过于工具化？如何让每个角色都有其存在的必要性？

### 5.2 故事结构优化

**期望反馈方向**：
1. 故事的整体结构是否合理？是否有逻辑漏洞？
2. 叙事节奏是否合适？是否有过于拖沓或过于急促的部分？
3. 叙事诡计的使用是否恰当？是否有过度使用或使用不当的地方？
4. 多周目设计是否能够有效支撑故事？玩家是否有足够的探索动力？

### 5.3 主题表达优化

**期望反馈方向**：
1. 主题是否足够清晰？是否有需要更突出的核心主题？
2. 主题的表达方式是否恰当？是否过于说教或过于隐晦？
3. 游戏是否能够引发玩家的思考？是否有足够的深度？
4. 游戏的"教育意义"和"娱乐性"是否平衡？

### 5.4 玩家体验优化

**期望反馈方向**：
1. 玩家体验是否流畅？是否有过于困惑或挫败的地方？
2. Meta元素的使用是否恰当？是否能够达到预期效果？
3. 游戏的难度曲线是否合理？是否有过于简单或过于困难的部分？
4. 玩家是否能够从游戏中获得足够的满足感和反思？

## 六、补充说明

### 6.1 游戏目标受众

- **主要受众**：喜欢悬疑推理、叙事诡计、meta元素的玩家
- **次要受众**：对心理惊悚、社会议题感兴趣的玩家
- **预期体验时长**：单周目10-15分钟，完整体验需要多周目（预计总时长2-3小时）

### 6.2 开发限制

- **技术限制**：文字冒险游戏，主要依赖文本和简单的UI交互
- **内容限制**：需要平衡暴力内容的呈现，避免过度渲染
- **时间限制**：项目处于开发阶段，剧本仍在完善中

### 6.3 参考作品

- **心跳文学部（Doki Doki Literature Club）**：Meta元素、打破第四面墙
- **谁是劳拉（Who is Laura）**：名字与身份的模糊、叙事诡计
- **David Lynch 作品**：非线性叙事、模糊的现实与梦境边界

## 七、期望的反馈形式

### 7.1 反馈方式

- **书面反馈**：可以针对上述问题提供详细的文字回复
- **讨论形式**：可以安排线上或线下讨论，深入探讨具体问题
- **分阶段反馈**：可以分阶段提供反馈，先针对人物塑造，再针对故事结构等

### 7.2 反馈重点

- **优先关注**：人物动机的合理性、故事结构的逻辑性、主题表达的清晰度
- **次要关注**：细节优化、语言风格、玩家体验的微调

### 7.3 反馈时间

- **期望时间**：根据您的安排，可以在1-2周内提供初步反馈
- **后续跟进**：可以根据反馈进行修改，然后进行第二轮咨询
