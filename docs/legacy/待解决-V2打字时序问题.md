# 待解决问题

## 问题描述

V2 系统中，文本和选项同时出现，而不是先显示文本打字完成后再显示选项。

### 现象
- 文本和选项一起出现
- 文本缓慢打字

### 期望
- 文本首先缓慢打字
- 文本打完字后，才出现选项

## 调试信息

从日志分析，`onLineComplete` 在页面加载后约 974ms 就被调用了，说明 TypingTextV2 组件几乎立即触发了 `complete` 事件，而不是等待打字完成。

```
[1770630938343] shouldShowLine index=0, type=narration, status=active, show=true
[1770630938343] shouldShowLine index=1, type=choice, status=pending, show=false
[1770630939317] onLineComplete called  ← 仅 974ms 后
```

## 可能原因

1. `TypingTextV2` 的 `props.nodes` 为空数组，导致立即调用 `complete()`
2. `script-compiler` 的文本解析逻辑有问题，没有正确生成 `TextNode[]`
3. `NarrationLineV2` 传递给 `TypingTextV2` 的 `nodes` prop 不正确

## 下一步调试

1. 在 `TypingTextV2.typeNext()` 中添加日志，检查 `props.nodes` 的内容
2. 检查 `script-compiler.ts` 的 `compileTextBlock()` 函数
3. 检查 `NarrationLineV2` 如何传递 `line.nodes` 给 `TypingTextV2`
