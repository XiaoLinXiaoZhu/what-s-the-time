# 剧本格式

## 概述

游戏剧本使用 Markdown 格式编写，由解析器（`src/parser/`）解析为结构化数据，再由引擎（`src/engine/`）驱动游戏流程。

## 文件结构

```markdown
---
id: SEGMENT-ID
time: HH:MM | START | "*"
description: 片段描述
loop: A | B | C | "*"
unlockFlags: [FLAG1, FLAG2]
---

行类型
内容
```

## Frontmatter 字段

| 字段 | 必需 | 说明 |
|------|------|------|
| `id` | 是 | 片段唯一标识，用于 jump 跳转 |
| `time` | 是 | 对应的关键时间点，`START` 表示开场，`*` 表示通配 |
| `description` | 否 | 片段描述（开发用） |
| `loop` | 否 | 所属【Loop系统】阶段 |
| `unlockFlags` | 否 | 前置解锁条件（flag 数组） |

## 行类型

### narration（旁白）
```
narration
风声。尖锐的、撕裂空气的风声。
```

### dialogue（对话）
```
dialogue 角色名
对话内容。可以有多行。
```
省略角色名则为【主角】内心独白。

### choice（选择）
```
choice
- [选项文本](#scope-id)
- [选项文本](#OTHER-SEGMENT-ID)
```
`#scope-id` 跳转到同文件内的 scope；`#SEGMENT-ID` 跳转到其他片段。

### input（输入）
```
input
提示文本 (HH:MM)
```
用于【玩家】输入时间。

### command（命令）
```
command
jump TARGET
setFlag FLAG_NAME
unsetFlag FLAG_NAME
end 结束文本
```

## Scope（子片段）

用 `# scope-id` 定义子片段，支持选择分支和循环跳转：

```markdown
choice
- [继续](#next)
- [返回](#loop)

# next
narration
继续的内容。

# loop
narration
返回循环。
command
jump SEGMENT-ID
```

## 文本格式标记

| 语法 | 效果 |
|------|------|
| `{red}文本{/red}` | 红色文本 |
| `{italic}文本{/italic}` | 斜体 |
| `**文本**` | 加粗 |
| `*文本*` | 斜体（Markdown 标准） |
| `{delay:N}` | 延迟 N 秒 |
| `{typewriter}a,b,c{/typewriter}` | 打字机效果逐字显示 |
