# 建筑师不是导游

## 核心思维

**不要设计"玩家体验线"。那会杀了这条鱼。**

取而代之，去创造一堆【记忆的房间】。每一个房间都对应一个【玩家】可以输入的【时间锚点】。你的工作不是"安排路线"，而是"布置房间"。

【玩家】自己会成为那条线。他们会从一个房间走到另一个房间，自己串联起一切。

## 鞋盒比喻

这个游戏玩起来，不是"看小说"，也不是"逛博物馆"。

**它感觉像是，在你新搬进的房子的地板下面，发现了一个旧的鞋盒。**

鞋盒里没有说明书。只有一堆零散的东西：几张褪色的照片、一封没写完的信、一张揉皱的电影票、一把没有锁的钥匙。你一张一张地看。起初你觉得是爱情故事。然后你感到寒意——你不是在整理故事，你是在**侵入一个秘密**。

## 【记住疼痛而非诡计】

这个项目最可能死于"变得太聪明，而忘了疼痛"。

当你过于专注于"盒子"的精巧时，你可能会忘了盒子里装着的是什么：
- 一个男孩在【03:15】的恐惧
- 【A】从天台坠落时，身体里骨头碎裂的声音
- 【B】意识到自己只是个替身时的空洞
- 【E】戴上手套时，指尖的冰冷

**如果玩家通关后感觉是"我真聪明，解开了复杂谜题"而不是"我说不出话"，那鱼就死了。**

怎么避免？在写作的每一天，都去【03:15】那个房间里坐一会儿。感受那个黑暗的房间。那是你的锚。
