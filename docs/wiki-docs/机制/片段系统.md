# 片段系统

## 概念

游戏内容以"片段"（Segment）为单位组织。每个片段对应一个【记忆的房间】，由 Markdown 文件定义。

## 片段文件格式

片段文件存放在 `src/data/segments/` 目录下，使用【剧本格式】编写：

```markdown
---
id: SEGMENT-ID
time: HH:MM | START | "*"
description: 片段描述
loop: A | B | C | ...
unlockFlags: [FLAG1, FLAG2]
---

narration
旁白文本

dialogue 角色名
对话文本

choice
- [选项A](#scope-a)
- [选项B](#OTHER-ID)

command
jump TARGET | setFlag FLAG | end
```

## 片段编号规则

| 编号前缀 | 阶段 | 示例 |
|----------|------|------|
| P00 | 【A案】前（普通恋爱期） | P00-01（正常早晨）、P00-02（谈一谈） |
| P01 | 【A案】当天 | P01-01（约楼顶消息）~P01-08（E远视目击） |
| P02 | 【A案】善后 | P02-01（问话）~P02-03 |
| P03 | 【B】线日常 | P03-01~P03-04 |
| P04 | 【B案】酝酿 | P04-01~P04-03 |
| P05 | 【B案】当天 | P05-01~P05-08 |
| P06 | 【B案】善后 | P06-01~P06-04 |
| P07 | 真相揭露 | P07-01~P07-02 |
| P08 | 【E】线 & 共犯 | P08-01~P08-03 |
| P09 | 终局 | P09-01~P09-02 |

## 版本控制

同一【时间锚点】可能有多个版本的片段：
- **假闪回版**：首次命中时展示的模糊版本
- **真相版**：满足【两问一锁】条件后解锁的完整版本
- 通过 `unlockFlags` 字段控制哪个版本展示给【玩家】
