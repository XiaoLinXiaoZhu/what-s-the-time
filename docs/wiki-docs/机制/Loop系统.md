# Loop系统

## 概述

游戏通过多个 Loop（周目）逐步揭示真相。每个 Loop 为【玩家】提供一组可访问的【时间锚点】，通过 flag 控制深层内容的解锁。

## Loop 总览

| Loop | 代号 | 目标 | 玩家认知状态 |
|------|------|------|-------------|
| A | 完全伪日常 | 只暴露【B】线日常，不提【A】 | "这是校园恋爱" |
| B | B消失后的错位 | 让"空位"先吓人 | "她去哪了？" |
| C | 假闪回与旧案影子 | 引出旧案，代词模糊 | "当年是意外？" |
| D | A真相 & 初步接触B案 | 通过【两问一锁】拆开 | "等等……两次？" |
| E | B真相 & E共犯 | 完整真相揭露 | "所有人都在骗我" |
| F | E顶罪 & 后事 | 善后片段 | 绝望与虚无 |
| G | 【Meta突破】终局 | 现实时间折叠 | "What's the time?" |

## Loop 结束条件

每个 Loop 通过 flag 机制控制进入下一阶段：
- **Loop A → B**：【玩家】看过 08:05 和 12:10，确认"【B】存在且关系升温"
- **Loop B → C**：【B】失踪被确认，【玩家】开始追问
- 以此类推，关键信息用 flag 控制

## 时间保护

如果【玩家】在早期 Loop 乱输敏感时间（如 10:37），统一导向"记忆空白片段"（BLANK），不允许过早进入闪回。

## 与【片段系统】的关系

每个 Loop 内部是"若干时间输入 → 对应若干片段"。片段通过 `loop` 字段标记所属阶段，通过 `unlockFlags` 控制前置条件。
