# 文风质量保证指南

## 核心理念

### 一、文风的目标

本游戏的文风旨在建立"人间失格"氛围，其最终体验并非解谜的快感或情感的宣泄，而是类似于读完《人间失格》后，那种对人性深渊的凝视所带来的、无法言说且挥之不去的压抑与虚无感。

**核心特征**：
- **拒绝提供简单的道德答案**：不进行说教，不提供审判，不寻求意义
- **通过文本本身的力量**：让玩家体验从"甜蜜日常"到"发现真相"再到"自我怀疑"的情感历程
- **最终落点**：无法言说、挥之不去的压抑、虚无和对人性深渊的凝视

### 二、文风在游戏叙事中的作用

文风服务于游戏的四个情感阶段，每个阶段都有明确的情感目标：

**前期**：让玩家代入主角，体验类似于校园恋爱题材的作品，但带有病态控制的色彩。玩家与主角的情感锚点是"和B的情感"，但这种情感从一开始就带有病态控制的色彩——玩家代入的是一个看似深情、实则正在将恋人"物品化"的角色的视角。文风需要营造"甜蜜中的诡异"，让玩家在享受甜蜜的同时，隐约会感到一丝不协调和控制感。

**中期**：让玩家与主角共情"B怎么死了？我是受害者"这样的共情路径。玩家需要扮演主角证明自己的无罪和伤痛。文风需要强化主角的"受害者形象"，让玩家相信主角是无辜的。

**中后期**：让玩家逐渐发现主角的异常，跟随这个异常重新探索新的时间片段以及整理整个故事的真实脉络。玩家的情绪从"我爱的人怎么死了？"转变为"我们（我和主角）的爱，是不是从一开始就错了？"这种自我怀疑，这种对自己前期情感投入的否定，是通往"人间失格"式虚无感的入口。文风需要逐步揭示主角的真实面貌，让玩家意识到"创伤后遗症"只是面具。

**后期**：meta元素让玩家扮演主角执行杀人，和C（理想化的"她"）跳出游戏框架，询问主角时间。此时玩家的现在时间和游戏的虚拟时间达成统一。文风需要通过meta元素，将"人间失格"的体验从观察主角，转变为反思自我。玩家在凝视深渊之后，发现深渊也在凝视自己。

### 三、文风的核心特征

为了实现上述目标，文风需要具备以下四个核心特征，这四个特征构成了"四大支柱"：

1. **不可靠的第一人称**：主角的叙述是经过自我美化和扭曲的"他的真实"，他说的未必是谎言，但永远是经过自我美化和扭曲的"他的真实"。他通过"辩解"来维护自己的"受害者形象"，将责任转嫁给他人。

2. **疏离感**：主角无法真正融入世界，他的一切行为都是在"扮演"一个正常人。他对周围的一切都有一种抽离的、冷漠的观察感。这种疏离感让玩家感受到主角与世界的隔阂，也为后续的真相揭露埋下伏笔。

3. **耻感与自我厌恶**：尽管主角不断辩解，但在其内心深处，是对自己存在的强烈"耻感"和厌恶。这种情绪不是以外显的忏悔，而是以自嘲、自我贬低的形式出现。这种内在的冲突，让主角的形象更加复杂和真实。

4. **语言的"表演性"**：文风是颓废的，但绝不粗糙。它有一种经过精心打磨的、病态的美感。用词讲究，句式从容，即使在描述最不堪的情感时，也保持着一种奇异的优雅。这种"表演性"正是主角"完美面具"的体现。

这四个特征将在第一部分"写作原则"中详细展开，并提供具体的执行标准。

## 第一部分：写作原则（四大支柱）

### 原则一：不可靠的第一人称——“我的辩解”

太宰治的主角总是在"辩解"，向一个不存在的法官或读者解释"我为何如此"。他说的未必是谎言，但永远是经过自我美化和扭曲的"他的真实"。

**执行标准**：
- **多用自问自答**： "我为何要那么做呢？大概是因为…"、"旁人看来或许很奇怪吧。但我只是…"
- **淡化恶意，强调"懦弱"与"恐惧"**：主角在叙述自己的恶行时，不会说"我出于恶意"，而会说"我因为害怕失去，才做了那件无可奈何的事。"
- **对他人行为的病态解读**：将B的觉醒解读为"她不再纯粹了"，将D的关心解读为"多管闲事的窥探"。

### 原则二：疏离感的营造——“我与世界之间隔着一层薄膜”

主角无法真正融入世界。他的一切行为都是在"扮演"一个正常人。他对周围的一切都有一种抽离的、冷漠的观察感。

**执行标准**：
- **感官描写的"失焦"**：描写环境时，多用模糊、失焦的词语。不说"阳光明媚"，而说"某种刺眼的光线从窗外渗进来，让我感到一阵眩晕。"
- **将情感"物体化"**：不直接说"我很难过"，而说"一种类似悲伤的、沉重的东西，盘踞在我的胸口。"
- **引用他人的"正常"**：频繁引用他人的话语或行为，来反衬自己的格格不入。"'普通人'在这种时候，似乎应该会哭吧。但我一滴眼泪也流不出来。"

### 原则三：耻感与自我厌恶——“我是个怪物”

尽管主角不断辩解，但在其内心深处，是对自己存在的强烈"耻感"和厌恶。这种情绪不是以外显的忏悔，而是以自嘲、自我贬低的形式出现。

**执行标准**：
- **突兀的自我嘲讽**：在一段看似正常的叙述后，突然插入一句自我否定的内心独白。例如，在描写完自己如何体贴B之后，内心独白："（看呐，一个多么完美的、体贴的男友。）"
- **对赞美的恐惧**：当被人夸奖时，主角的内心反应不是喜悦，而是恐慌，因为"他们称赞的那个'我'，根本不存在。"

### 原则四：语言的"表演性”——精致的颓废

太宰治的文风是颓废的，但绝不粗糙。它有一种经过精心打磨的、病态的美感。用词讲究，句式从容，即使在描述最不堪的情感时，也保持着一种奇异的优雅。

**执行标准**：
- **避免口语化、网络化的词语**：除非是引用他人对话，否则主角的内心独白应保持书面语的精致。
- **多用比喻，尤其是自然或物理比喻**： "我的谎言像藤蔓一样，缠绕住她，也缠绕住我自己。"、"这份爱，像一块冰，被我死死攥在手心，既不愿它融化，又被冻得生疼。"

## 第二部分：技术规范（交互式文本编写）

本部分提供具体的技术实现方法，帮助将第一部分"写作原则"落地到实际的文本编写中。

### 一、文本类型区分

**核心原则：说出口的 vs 心里想的**

**重要规范**：所有文本应当包裹为类似galgame的对话和内心独白形式，主角的话统一使用 dialogue 类型，且无 character 属性。

#### dialogue（无 character 属性）

- **主角说出口的话**：所有主角的对话、自言自语、失控喊叫、低语等
- 当下失控、下意识说出口的话（痴语、喊叫、低语）
- 主角的内心独白如果以"说出口"的形式呈现，也应使用 dialogue

**示例**：
- `{red}不……{/red}{br}停下……` ← 恐慌中的喊叫
- `{red}……{typewriter}10:37{/typewriter}{/red}{br}——手表上的指针，停在了这一刻。` ← 低声念出关键信息

#### narration

- 事后回看、分析、解释性的内心独白（纯"脑内声音"）
- 感官描写、环境渲染、情绪物化
- 自问自答式的辩解与反思（如果是以"脑内声音"呈现）

**示例**：
- `是吗？{br}现在是……这个时间吗？` ← 回顾式的自我质疑

**判断技巧**：如果这句话"有声音"（哪怕是压抑的、小声的），用 dialogue；如果是"脑内声音"，用 narration。

### 二、标签使用规范

#### 2.1 {br} 换行策略

**{br} 的三大功能**：
1. **视觉换行**：让文本分行显示，形成上下对照的视觉节奏
2. **自动流淌**：内容会自动播放，不需要玩家点击
3. **呼吸节奏**：配合 delay 使用，控制信息展开的速度（参见 2.2 {delay} 节奏控制）

**何时用 {br}（段内换行）**：
- 同一思绪流、同一场景的连续描述
- 反复、纠结、追问的句子（详细技法参见 3.1 反复/纠结/对比表现技法）

**示例**：
```
时间不对。{br}{delay:0.8}
哪里不对？{br}{delay:0.8}
我试图抓住什么，但只有虚无。
```

**何时拆成独立 item（需要玩家点击）**：
- **关注点切换**：从视觉转向听觉、从回忆转向现实
  - 例如：`模糊的红色...吞噬一切。` （视觉） → 下一个 item：`尖叫声...` （听觉）
- **情绪节点**：需要玩家停顿、消化、感受的关键时刻
- **重要信息揭示**：让玩家"点一下"来确认接收

#### 2.2 {delay} 节奏控制

**核心原则**：少即是多，精准即有力

**标准时长**：
- **0.5 秒**：文本的稍微停顿，模拟正常人对话的自然流动感（"对，{delay:0.5}一定是梦"）
- **0.8~1.0 秒**：自然对话停顿、句间呼吸（推荐范围）
- **1.0~1.2 秒**：段落转折、情绪沉淀
- **1.5 秒**：极限，仅用于重大情绪冲击点（基本不用）

**重要规范**：
- ⚠️ **delay 不应超过 1.5 秒**，大多数情况下应控制在 1.0 秒以内
- ⚠️ **0.5 秒用于自然停顿**，模拟正常人对话的流动感，让文本有呼吸节奏
- ⚠️ **自然对话停顿使用 0.8~1.0 秒**，保持对话和叙事的自然顺畅感
- ⚠️ **逐字强调应使用 {typewriter} 标签**，而非 delay（参见 2.3 {typewriter} 逐字强调）

**禁止滥用**：
- ❌ 连贯动作描述中不加 delay：`像一把刀刺破了我的意识` （一气呵成）
- ❌ 超过 1.5 秒的 delay 会让节奏拖沓、破坏沉浸感

**{br}{delay:...} vs {delay:...}{br}**：
- ✅ **推荐**：`时间不对。{br}{delay:0.8}这个念头...`
  - 换行发生在前，玩家先看到光标移到下一行，知道"还有话"，然后等待文字出现
  - 更自然，有"思考后继续说"的感觉
- ❌ **不推荐**：`时间不对。{delay:0.8}{br}这个念头...`
  - 先停顿，再换行，感觉割裂

#### 2.3 {typewriter} 逐字强调

**功能**：让文字缓慢逐字打出，营造"凝视""回忆""强调"的效果

**重要规范**：逐字强调推荐使用 {typewriter} 标签

**适用场景**：
1. **关键时间信息**（推荐使用）：
   - `{red}{typewriter}10:37{/typewriter}{/red}` ← 核心时间锚点（时间信息标红规范参见 2.4 {red} 视觉强调）
   - `{red}{typewriter}08:05{/typewriter}{/red}` ← 重要回忆时刻
   
2. **反复出现的关键词**：
   - `{red}{typewriter}时间不对{/typewriter}{/red}` ← 第二次出现"时间不对"时用 typewriter 强调执念

3. **精神状态的核心词**：
   - 例如：`{typewriter}她的名字{/typewriter}` / `{typewriter}那个约定{/typewriter}`

**使用技巧**：
- typewriter 后通常加一个短 delay（0.6~0.8秒），让"缓慢打字"的效果被玩家充分感知
  - `{red}{typewriter}08:05{/typewriter}{/red}{delay:0.8}……`
- 与 {red} 组合使用，突出时间信息的重要性

**禁止滥用**：
- 一个段落最多 1~2 处 typewriter，否则失去特殊性
- 不用于普通对话或描述
- **注意区分**：逐字强调用 typewriter，自然停顿用 delay:0.5（参见 2.2 {delay} 节奏控制）

#### 2.4 {red} / {bold} 视觉强调

**{red}**：
- 危险、血腥、时间、执念
- **重要规范：所有时间相关信息必须标红或做出其他特殊处理，便于玩家识别**
  - 时间点：`{red}10:37{/red}` / `{red}{typewriter}10:37{/typewriter}{/red}`
  - 时间相关描述：`{red}那个瞬间{/red}` / `{red}时间不对{/red}`
- 例如：`{red}不……{/red}` / `{red}{typewriter}10:37{/typewriter}{/red}`

**{bold}**：
- 关键念头第一次出现时的强调
- 例如：`{bold}时间不对。{/bold}` ← 核心念头首次浮现

**组合使用**：
- `{red}{typewriter}时间不对{/typewriter}{/red}` ← 红色 + 缓慢打字 = 最强调（时间信息推荐使用此组合）

#### 2.5 {italic} / {blur} / {strike} 特殊效果

**{italic}**：
- 斜体文本，用于强调或特殊语气
- 适合用于：内心独白中的强调、回忆中的关键片段、对话中的特殊语气
- 例如：`我{italic}必须{/italic}做出决定。` ← 强调"必须"
- 组合使用：`{red}{italic}关键信息{/italic}{/red}` ← 红色斜体，双重强调

**{blur}**：
- 模糊文本，用于表示记忆模糊、不确定的状态
- **重要用途**：表示主角记忆的不可靠性，暗示记忆被篡改或模糊
- 例如：`时间是……{br}{delay:1}{blur}10:37{/blur}……{br}应该吧……` ← 记忆模糊，不确定
- 与 {red} 组合：`{red}{blur}那个瞬间{/blur}{/red}` ← 模糊但重要的时间信息

**{strike}**：
- 删除线文本，用于表示被篡改的记忆、修正的内容
- **重要用途**：在播放被篡改的记忆时，让文本出现划线修正，暗示记忆的不可靠性
- 例如：`{strike}旧的想法{/strike} {red}新的想法{/red}` ← 记忆被修正
- 组合使用：`{strike}{red}错误的时间{/red}{/strike} {red}{typewriter}10:37{/typewriter}{/red}` ← 修正后的时间信息

**使用技巧**：
- {blur} 和 {strike} 都服务于"不可靠的第一人称"原则，强化主角记忆的不可靠性
- {italic} 用于强调，但不应过度使用，否则失去特殊性
- 这些标签可以组合使用，但要注意不要过度复杂化

#### 2.6 {systemTime} / {animateText} 动态内容

**{systemTime}**：
- 系统当前时间（HH:MM格式，实时更新）
- **重要用途**：用于meta元素，将"叙事时间 / 游戏时间 / 现实时间"折叠到一起
- 例如：`现在是 {systemTime}。` ← 显示玩家设备上的真实时间
- 在最终meta环节使用：`不对，现在是 {systemTime}，对吧？` ← 打破第四面墙，将玩家拉入责任链
- **使用规范**：仅在最终meta环节使用，保持冲击力

**{animateText:string1|string2|string3}**：
- 动画文本，每0.5秒在多个字符串之间切换
- 用于状态切换、动态显示、营造不安感
- 例如：`不，是 {animateText:它|她|他|ta|祂}` ← 不断变化的称谓
- 例如：`{animateText:…|……|………}` ← 营造不断思虑的焦躁感
- **使用技巧**：
  - 用于表示主角精神状态的不稳定
  - 用于表示系统状态的动态变化
  - 不应过度使用，否则会分散注意力

**禁止滥用**：
- {systemTime} 仅在最终meta环节使用，不要提前使用
- {animateText} 不应用于关键信息，因为动态切换会让玩家难以阅读

### 三、表现技法

#### 3.1 反复/纠结/对比表现技法

**原则**：分行显示才有张力

当主角在两个念头之间徘徊、反复追问、对比矛盾时，必须用 {br} 让句子分行（参见 2.1 {br} 换行策略）：

**❌ 错误示例（一行显示，失去纠结感）**：
```
是她的声音吗？还是……我的？我分不清。
```

**✅ 正确示例（分行显示，形成上下对照）**：
```
是她的声音吗？{br}
还是……我的？{br}
像一部坏掉的电影，声音和画面错位。
```

**适用场景**：
- 自我追问：`时间不对。{br}哪里不对？`
- 对比怀疑：`现在是……这个时间吗？{br}还是说，那只是某一次……坠落的时间？`
- 递进强调：`只有风声，{br}只有红色，{br}只有……`
- 反复尝试：`我试图拼凑，{br}试图抓住，{br}但只有虚无。`

#### 3.2 比喻与意象策略

**原则**：画面感优先于概念

**❌ 错误示例（概念先行，画面感弱）**：
```
模糊的红色。{br}
像颜料一样，在视野里晕开...
```
"模糊的红色"过于抽象，玩家没有画面感。

**✅ 正确示例（比喻前置，画面感强）**：
```
模糊的红色，像颜料一样。{br}
在视野里晕开，蔓延，吞噬一切。
```
立刻给玩家"颜料"的具象画面，然后展开动作。

**适用场景**：
- 感官描写（视觉、听觉、触觉）
- 情绪物化（"像一把刀""像颜料""像冰"）

#### 3.3 精简与节奏的平衡

**原则**：去除冗余，保留必要的呼吸

叙事应该流畅有张力，应该按照叙事的节奏一张一弛。

**需要精简的**：
- 重复的追问（三个变两个）
- 冗余的形容词（"那个时间，那个瞬间" → "那个瞬间"）
- 过长的 delay（参见 2.2 {delay} 节奏控制，标准时长不应超过 1.5 秒）

**需要保留的**：
- 反复强调的核心词（"时间不对"出现两次，第二次用 typewriter）
- 递进的"只有"结构（风声/红色/...）
- 关键情绪节点的停顿

**示例对比**：

❌ **拖沓版本**：
```
一切都应该停在那个完美的早晨……{delay:1.5}{red}08:05{/red}……{br}
那个时间，那个瞬间，一切都还是……{delay:1.2}完美的。
```

✅ **精简版本**：
```
一切都应该停在那个完美的早晨……{br}
{red}{typewriter}08:05{/typewriter}{/red}{delay:0.8}……{br}
那个瞬间，一切都还是……{delay:0.8}完美的。
```

### 四、交互设计

#### 4.1 choice 选项设计

**原则**：选项出现前必须有自然的引导句，保持交互感觉和对话/叙事的自然顺畅感

**重要规范**：适当增加 choice 选项以提升沉浸感，让玩家主动参与叙事。

**选项设计原则**：
- **自然过渡**：选项出现前必须有自然的引导句，避免突兀
- **提升参与感**：在关键决策点、情感节点、认知确认点设置选项
- **保持节奏**：选项不应过于密集，也不应过于稀疏，需与叙事节奏匹配

**❌ 错误示例（突兀）**：
```
我想不太清楚。大概，也不需要想得那么清楚。
或者……我该承认些什么？

[选项出现]
```
"我该承认些什么"过于抽象，玩家不知道要"承认"什么。

**✅ 正确示例（自然过渡）**：
```
我想不太清楚。大概，也不需要想得那么清楚。
但总得有个答案。哪怕，只是说给自己听。

[选项出现]
```
"总得有个答案"明确告诉玩家：接下来要做决定了。

**选项类型**：
- **虚假选择**：体现玩家意图与主角行为的冲突。玩家选A，主角却做了B，并给出内心独白解释"我为什么不能那么做"。这强化了主角的"不可靠叙述者"特征，让玩家意识到自己并没有真正的控制权。
- **认知选择**：确认玩家的理解，用于关键信息验证。例如，在后期提问"杀死她的是谁？"，选项有H和"我"，选错会有特殊反馈。这帮助玩家确认自己对真相的理解。
