## 中间格式理解

### 一、中间格式概述

编剧使用**自然小说格式**创作，这是一种完全自然的Markdown格式，没有任何技术标记。编剧只需要：

- 使用 `*斜体*` 表示内心独白/旁白
- 使用 `**粗体**` 表示强调
- 使用自然对话格式 `角色名: "对话内容"`
- 自然换行，不需要关心技术细节

**你的任务**：理解编剧的创作意图，根据情节和强调重点，手动转换为程序格式。

### 二、中间格式结构

#### 片段元数据（YAML Front Matter）
```markdown
---
片段ID: P1-DTL-T0805-Breakfast
时间: 08:05
阶段: P1
解锁条件: FLAG_Game_Start
描述: 阶段1 - 早餐
作用: 建立"虚构恋爱史"的开端；情感绑定。
---
```

**转换规则**：
- `片段ID` → `id: 'P1-DTL-T0805-Breakfast'`
- `时间` → `time: '08:05'`
- `阶段` → `loop: 'P1'`
- `解锁条件` → `unlockFlags: ['FLAG_Game_Start']`（多个条件用逗号分隔，转换为数组）
- `描述` → `description: '阶段1 - 早餐'`

#### 文本内容

编剧使用自然格式创作，你需要：
1. **理解情节**：理解这段文本在游戏中的作用
2. **识别重点**：识别需要强调的内容（时间、关键念头、危险等）
3. **判断节奏**：根据情节判断节奏停顿的位置和时长
4. **转换格式**：转换为程序格式

## 转换原则

### 一、理解文风原则

在转换前，你必须理解游戏的文风原则（参考《编剧创作指南》）：

1. **不可靠的第一人称**：主角的叙述是经过自我美化和扭曲的
2. **疏离感**：主角无法真正融入世界，对周围的一切都有抽离的观察感
3. **耻感与自我厌恶**：主角内心深处有强烈的"耻感"和厌恶
4. **语言的"表演性"**：文风是精致的颓废，用词讲究，句式从容

### 二、理解情节意图

在转换时，你需要理解编剧的创作意图：

- **情感目标**：这段文本要达到什么情感效果？
- **节奏控制**：哪里需要停顿？哪里需要一气呵成？
- **视觉强调**：哪些内容需要特殊处理（时间、关键念头、危险等）？
- **交互设计**：是否需要选择分支？是否需要时间输入？

### 三、手动转换原则

**重要**：这不是自动化转换，而是**手动理解**和**艺术性转换**。

你需要：
- ✅ 理解情节和创作意图
- ✅ 根据文风原则判断节奏和强调
- ✅ 手动转换为程序格式
- ❌ 不要机械地转换，要理解编剧的创作意图

## 文本类型判断

### 一、判断标准

**核心原则：说出口的 vs 心里想的**

#### narration（内心独白/旁白）

编剧使用 `*斜体*` 标记的内容，对应 `narration` 类型。

**判断标准**：
- 主角的内心想法、回忆、分析、解释
- 感官描写、环境渲染、情绪物化
- 自问自答式的辩解与反思（如果是"脑内声音"）

**转换规则**：
```markdown
*早晨的阳光透过梧桐叶片的缝隙，像某种精致的花边装饰。*
```
↓
```typescript
{ type: 'narration', text: '早晨的阳光透过梧桐叶片的缝隙，像某种精致的花边装饰。' }
```

#### dialogue（主角说出口的话）

编剧直接写的对话内容（没有角色名），对应 `dialogue` 类型（无character属性）。

**判断标准**：
- 主角说出口的话（包括自言自语、失控喊叫、低语等）
- 当下失控、下意识说出口的话（痴语、喊叫、低语）

**转换规则**：
```markdown
她指着街角那家新开的包子店，眼睛亮晶晶的。"我们去试试那个吧？听说皮薄馅大，很香的。"
```
↓
```typescript
{ type: 'dialogue', text: '她指着街角那家新开的包子店，眼睛亮晶晶的。"我们去试试那个吧？听说皮薄馅大，很香的。"' }
```

**注意**：如果主角表示沉默，应该使用 `dialogue` 类型，表现为"……"或类似形式。

#### dialogue（其他角色对话）

编剧使用 `角色名: "对话内容"` 格式的内容，对应 `dialogue` 类型（有character属性）。

**转换规则**：
```markdown
B: "她眨了眨眼，像是被什么小东西绊了一下。"可是……偶尔试试也没关系吧？"
```
↓
```typescript
{ type: 'dialogue', character: 'B', text: '她眨了眨眼，像是被什么小东西绊了一下。"可是……偶尔试试也没关系吧？"' }
```

**注意**：如果角色有动作，应该在主角的 `narration` 中描述主角看到的动作。

### 二、常见场景判断

| 编剧格式 | 程序格式 | 说明 |
|---------|---------|------|
| `*内心想法*` | `{ type: 'narration' }` | 纯"脑内声音" |
| `"对话内容"`（无角色名） | `{ type: 'dialogue' }` | 主角说出口的话 |
| `角色名: "对话内容"` | `{ type: 'dialogue', character: '角色名' }` | 其他角色对话 |
| `……`（主角的沉默） | `{ type: 'dialogue', text: '……' }` | 主角的沉默 |

## 节奏与停顿处理

### 一、判断原则

**核心原则**：根据情节和创作意图判断节奏，而不是机械地转换。

### 二、换行处理

编剧的自然换行表示节奏停顿，你需要根据情节判断：

1. **段内换行**（使用 `{br}`）：
   - 同一思绪流、同一场景的连续描述
   - 反复、纠结、追问的句子
   - 需要形成上下对照的视觉节奏

2. **拆分成独立 item**（需要玩家点击）：
   - **关注点切换**：从视觉转向听觉、从回忆转向现实
   - **情绪节点**：需要玩家停顿、消化、感受的关键时刻
   - **重要信息揭示**：让玩家"点一下"来确认接收

### 三、停顿时长判断

根据情节和创作意图判断停顿时长：

- **0.5 秒**：文本的稍微停顿，模拟正常人对话的自然流动感
- **0.8~1.0 秒**：自然对话停顿、句间呼吸（推荐范围）
- **1.0~1.2 秒**：段落转折、情绪沉淀
- **1.5 秒**：极限，仅用于重大情绪冲击点（基本不用）

**判断技巧**：
- 如果编剧在关键情绪节点换行，通常需要 `{br}{delay:0.8}` 或 `{br}{delay:1.0}`
- 如果编剧在反复追问时换行，通常需要 `{br}{delay:0.8}`
- 如果编剧在段落转折时换行，通常需要 `{br}{delay:1.0}`

### 四、转换示例

**编剧格式**：
```markdown
*早晨的阳光透过梧桐叶片的缝隙，像某种精致的花边装饰。*

*这是新学校的第一周。一切都应该是完美的。*
```

**转换判断**：
- 两段都是内心独白，可以合并为一个 `narration` item
- 中间有自然停顿，使用 `{br}{delay:0.8}`

**程序格式**：
```typescript
{
  type: 'narration',
  text: '早晨的阳光透过梧桐叶片的缝隙，像某种精致的花边装饰。{br}{delay:0.8}这是新学校的第一周。一切都应该是完美的。'
}
```

**编剧格式**：
```markdown
我看着她，心里涌起一种奇怪的情绪。不是爱——那种东西太虚假了。

是一种……占有欲？不对。是保护欲？也不对。

"你的胃不好，早上吃这么油腻的东西会不舒服的。"
```

**转换判断**：
- 这是主角的内心纠结，然后说出口的话
- 换行表示节奏停顿，需要 `{br}{delay:0.8}`
- 可以合并为一个 `dialogue` item，因为都是主角的思绪流

**程序格式**：
```typescript
{
  type: 'dialogue',
  text: '我看着她，心里涌起一种奇怪的情绪。不是爱——那种东西太虚假了。{br}{delay:0.8}是一种……占有欲？不对。是保护欲？也不对。{br}{delay:0.8}"你的胃不好，早上吃这么油腻的东西会不舒服的。"'
}
```

## 视觉强调处理

### 一、判断原则

编剧使用 `**粗体**` 表示强调，你需要根据情节判断如何转换：

1. **时间信息**：必须使用 `{red}{typewriter}HH:MM{/typewriter}{/red}` 组合
2. **关键念头**：使用 `{bold}...{/bold}`
3. **危险/执念**：使用 `{red}...{/red}`
4. **记忆模糊**：使用 `{blur}...{/blur}`
5. **记忆修正**：使用 `{strike}...{/strike} {red}...{/red}`
6. **逐字强调**：使用 `{typewriter}...{/typewriter}`

### 二、时间信息处理

**重要规范**：所有时间相关信息必须标红或做出其他特殊处理，便于玩家识别。

**判断标准**：
- 如果文本中提及时间（如"10:37"、"08:05"），必须使用 `{red}{typewriter}HH:MM{/typewriter}{/red}` 组合
- 如果文本中提及时间相关描述（如"那个瞬间"、"时间不对"），根据情节判断是否需要标红

**转换示例**：
```markdown
*我看着她的背影，想起了一个问题。*

***她今天露出的疲惫笑容，是从哪里学来的？***
```

**转换判断**：
- 这是关键问题，需要强调
- 但这不是时间信息，使用 `{bold}...{/bold}` 即可

**程序格式**：
```typescript
{
  type: 'narration',
  text: '我看着她的背影，想起了一个问题。{br}{delay:1.0}{bold}她今天露出的疲惫笑容，是从哪里学来的？{/bold}'
}
```

### 三、关键念头处理

**判断标准**：
- 如果编剧使用 `**粗体**` 标记关键念头第一次出现，使用 `{bold}...{/bold}`
- 如果关键念头反复出现，第二次可以使用 `{red}{typewriter}...{/typewriter}{/red}` 组合

### 四、危险/执念处理

**判断标准**：
- 如果文本涉及危险、血腥、执念等内容，使用 `{red}...{/red}`
- 根据情节判断是否需要逐字强调（`{typewriter}`）

### 五、记忆模糊/修正处理

**判断标准**：
- 如果文本表示记忆模糊、不确定，使用 `{blur}...{/blur}`
- 如果文本表示记忆被修正，使用 `{strike}...{/strike} {red}...{/red}`

## 交互元素实现

### 一、选择分支

**判断标准**：
- 如果编剧在文本中描述了选择场景，你需要判断是否需要实现选择分支
- 参考《编剧创作指南》中的"选项设计原则"

**实现格式**：
```typescript
{
  type: 'choice',
  choices: [
    {
      text: '选项文本1',
      lines: [
        // 选择后的内容
      ],
      setFlag: 'FLAG_NAME' // 可选
    }
  ]
}
```

### 二、时间输入

**判断标准**：
- 如果片段需要玩家输入时间，使用 `input` 类型

**实现格式**：
```typescript
{
  type: 'input',
  placeholder: '请输入时间（HH:MM格式）'
}
```

### 三、时间选择

**判断标准**：
- 如果片段需要根据输入的时间匹配不同的内容，使用 `timeChoice` 类型

**实现格式**：
```typescript
{
  type: 'timeChoice',
  choices: [
    {
      time: '08:05',
      lines: [
        // 匹配后的内容
      ],
      setFlag: 'FLAG_NAME' // 可选
    }
  ]
}
```

### 四、命令

**判断标准**：
- 如果需要在特定位置设置标记，使用 `command` 类型

**实现格式**：
```typescript
{
  type: 'command',
  command: 'setFlag',
  params: { flag: 'FLAG_NAME' }
}
```

## 转换工作流程

### 一、转换前准备

1. **阅读编剧文档**：理解文风原则和创作意图
2. **阅读片段设计文档**：理解片段在游戏中的作用
3. **理解情节**：理解这段文本要达到什么情感效果

### 二、转换过程

1. **解析元数据**：将YAML Front Matter转换为TypeScript对象属性
2. **理解文本**：理解编剧的创作意图和情节
3. **判断类型**：根据格式判断文本类型（narration vs dialogue）
4. **处理节奏**：根据情节判断节奏停顿的位置和时长
5. **处理强调**：根据情节判断视觉强调的方式
6. **生成代码**：生成符合 `ScriptSegment` 接口的代码

### 三、转换后验证

1. **格式验证**：检查代码是否符合 `ScriptSegment` 接口
2. **标签验证**：检查标签是否正确闭合
3. **节奏验证**：检查节奏是否合理
4. **强调验证**：检查强调是否恰当

| `{br}` | 换行、自动播放 | 段内换行、反复追问 |
| `{delay:X}` | 节奏停顿（秒） | 自然停顿、句间呼吸 |
| `{typewriter}` | 逐字强调 | 关键时间、核心词 |
| `{red}` | 红色文本 | 时间、危险、执念 |
| `{bold}` | 粗体 | 关键念头首次强调 |
| `{italic}` | 斜体 | 强调、特殊语气 |
| `{blur}` | 模糊文本 | 记忆模糊、不确定 |
| `{strike}` | 删除线 | 被篡改的记忆 |
| `{systemTime}` | 系统当前时间 | meta元素（仅最终环节） |
| `{animateText:str1\|str2}` | 动画文本 | 状态切换、不安感 |

### 常见问题

**Q: 如何判断使用 `narration` 还是 `dialogue`？**  
A: 如果编剧使用 `*斜体*`，使用 `narration`；如果是直接对话（无角色名），使用 `dialogue`；如果是 `角色名: "..."`，使用 `dialogue` with character。

**Q: 停顿标记应该放在哪里？**  
A: 根据情节判断。如果编剧在关键情绪节点换行，通常需要 `{br}{delay:0.8}` 或 `{br}{delay:1.0}`。

**Q: 时间信息必须使用红色和typewriter吗？**  
A: 是的，所有时间信息必须使用 `{red}{typewriter}HH:MM{/typewriter}{/red}` 组合，便于玩家识别。

**Q: 如何判断是否需要选择分支？**  
A: 根据情节判断。如果编剧在文本中描述了选择场景，参考《编剧创作指南》中的"选项设计原则"。

---

**重要**：转换不是机械的，而是**理解创作意图**和**艺术性转换**。你需要理解文风原则和情节，然后根据判断进行转换。
