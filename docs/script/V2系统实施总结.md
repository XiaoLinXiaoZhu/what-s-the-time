```typescript
parseNarration(text: string): NarrationLine {

}

parseNarration(text: string): NarrationTextLine {
  return { type: 'narration', nodes: parseText(text) }  // ✅ 已解析
}
```

**核心改进**:

**导航流程**:
```typescript
// 旧方式（V1）
handleChoice(choice) {
  // 直接插入内联行到 displayedLines
  displayState.displayedLines.splice(lineIndex + 1, 0, ...choice.lines)
}

// 新方式（V2）
async handleChoice(choice) {
  // 跳转到目标片段
  await navigationServiceV2.navigateToSegment(choice.targetSegments[0])
}
```

## ⚠️ 已知限制

### V2 系统当前限制
1. **独立状态管理**
   - NavigationServiceV2 和 ChoiceServiceV2 使用独立状态
   - 未集成到现有的 StateStore
   - **影响**: 需要手动初始化和管理 V2 状态

2. **SegmentRegistry 未集成**
   - 当前 V2 服务使用占位实现
   - 需要在应用启动时注册所有片段
   - **影响**: V2 导航无法加载未注册的片段

3. **类型转换工具未集成**
   - `typeConversion.ts` 创建完成但未在应用中使用
   - **影响**: 无法进行实际数据迁移

### 建议解决方案
1. **创建统一的 V2 状态管理**
   ```typescript
   // 扩展现有 StateStore 或创建新的
   export const stateStoreV2 = new StateStoreV2()
   ```

2. **集成 SegmentRegistry**
   ```typescript
   // 在 main.ts 或应用启动时
   import { initializeSegments } from '@/data/script'
   initializeSegments()  // 使用 V2 版本
   ```

3. **创建迁移工具脚本**
   ```bash
   # 创建命令行工具
   bun run migrate-to-v2
   ```
