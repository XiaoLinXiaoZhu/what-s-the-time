# 设计阶段报告：文档拆分与中间格式实现

## 一、设计成果

### 创建的文档

1. **《编剧创作指南.md》**
   - 目标读者：编剧
   - 核心内容：核心理念、写作原则、表现技法、中间格式规范、创作示例
   - 特点：专注创作指导，不包含技术细节

2. **《技术美术转换指南.md》**
   - 目标读者：技术美术
   - 核心内容：中间格式理解、转换规则、标签系统详解、文本类型判断、交互元素实现、转换示例
   - 特点：专注格式转换，包含所有技术细节

3. **《中间格式规范.md》**
   - 目标读者：编剧、技术美术
   - 核心内容：中间格式的完整规范和示例
   - 特点：作为两份文档的桥梁，提供统一的格式参考

### 中间格式设计

#### 设计原则
1. **自然性**：编剧可以像写小说一样创作
2. **语义化**：使用语义标记而非技术标记
3. **可转换性**：技术美术可以系统化转换
4. **可读性**：人类可读，便于审查和修改

#### 核心特性
- **文本类型标记**：`[内心]`、`[说]`、`[角色名]`
- **节奏与停顿标记**：`[停顿:短/中/长]`
- **视觉强调标记**：`[时间:HH:MM]`、`[强调]`、`[危险]`、`[模糊]`、`[修正]`、`[逐字]`
- **交互元素标记**：`[选择]`、`[标记:FLAG]`、`[输入时间]`、`[时间选择]`
- **片段元数据**：YAML Front Matter格式

## 二、文档拆分方案

### 拆分原则

1. **职责分离**：编剧专注创作，技术美术专注转换
2. **信息完整**：两份文档必须包含所有必要信息
3. **交叉引用**：必要时引用另一份文档
4. **独立可用**：每份文档可以独立使用

### 内容分配

#### 编剧文档包含
- ✅ 核心理念（文风目标、情感阶段、四大支柱）
- ✅ 写作原则（四大支柱的执行标准）
- ✅ 表现技法（反复/纠结、比喻、精简）
- ✅ 中间格式规范（如何使用语义化标记创作）
- ✅ 创作示例和最佳实践
- ❌ 技术标签细节（`{br}`, `{delay:0.8}`等）
- ❌ 程序结构细节（TypeScript类型定义）
- ❌ 转换规则（这些在技术美术文档中）

#### 技术美术文档包含
- ✅ 中间格式规范（理解编剧使用的标记）
- ✅ 转换规则（从中间格式到程序格式的映射）
- ✅ 标签系统详解（所有技术标签的使用规范）
- ✅ 文本类型判断（dialogue vs narration的判断标准）
- ✅ 交互元素实现（choice、timeChoice等的实现）
- ✅ 转换示例和工具建议
- ❌ 文风原则的详细解释（这些在编剧文档中）
- ❌ 创作技法（这些是编剧的职责）

### 交叉引用策略

1. **编剧文档**中引用技术美术文档：
   - "关于技术标签的具体实现，请参考《技术美术转换指南》"
   - "如需了解程序格式细节，请咨询技术美术"

2. **技术美术文档**中引用编剧文档：
   - "关于文风原则的详细说明，请参考《编剧创作指南》"
   - "如需理解创作意图，请参考编剧文档中的表现技法部分"

## 三、转换规则设计

### 映射表（中间格式 → 程序格式）

| 中间格式 | 程序格式 | 说明 |
|---------|---------|------|
| `[内心] ...` | `{ type: 'narration', text: '...' }` | 主角内心独白 |
| `[说] ...` | `{ type: 'dialogue', text: '...' }` | 主角说出口的话 |
| `[角色名] ...` | `{ type: 'dialogue', character: '角色名', text: '...' }` | 其他角色对话 |
| `[停顿:短]` | `{delay:0.5}` | 自然停顿 |
| `[停顿:中]` | `{delay:0.8}` | 句间呼吸 |
| `[停顿:长]` | `{delay:1.0}` | 段落转折 |
| `[时间:HH:MM]` | `{red}{typewriter}HH:MM{/typewriter}{/red}` | 时间信息 |
| `[强调] ...` | `{bold}...{/bold}` | 关键念头 |
| `[危险] ...` | `{red}...{/red}` | 危险/执念 |
| `[模糊]...[模糊]` | `{blur}...{/blur}` | 记忆模糊 |
| `[修正:旧] [强调:新]` | `{strike}旧{/strike} {red}新{/red}` | 记忆修正 |
| `[逐字] ...` | `{typewriter}...{/typewriter}` | 逐字强调 |
| `[选择] ...` | `{ type: 'choice', choices: [...] }` | 选择分支 |
| `[标记:FLAG]` | `{ type: 'command', command: 'setFlag', params: { flag: 'FLAG' } }` | 设置标记 |

### 转换流程

1. **解析中间格式**：读取Markdown文件，解析YAML Front Matter和语义标记
2. **应用转换规则**：根据映射表转换为程序格式
3. **生成TypeScript代码**：输出符合 `ScriptSegment` 接口的代码
4. **验证和优化**：检查格式正确性，优化标签使用

## 五、设计亮点

### 1. 语义化标记系统
- 使用 `[内心]`、`[说]` 等语义标记，而非 `{type: 'narration'}` 等技术标记
- 编剧无需理解技术细节，只需关注创作本身

### 2. 完整的转换映射
- 所有中间格式标记都有明确的程序格式对应
- 转换规则无歧义，便于技术美术系统化转换

### 3. 职责清晰分离
- 编剧文档专注创作指导
- 技术美术文档专注格式转换
- 中间格式规范作为桥梁

### 4. 可扩展性
- 中间格式可以轻松扩展新的标记类型
- 转换规则可以系统化应用
