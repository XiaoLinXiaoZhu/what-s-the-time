# 重新设计报告：完全自然的小说格式

## 一、重新设计背景

### 用户反馈

用户希望编剧能够**完全专注于创作**，不需要关心任何技术细节。用户提出的新方案：

1. 使用 `*ABC*` 表示旁白/内心独白（第一人称视角）
2. 使用 `**ABC**` 表示强调
3. 自然语言，只需要 `A:""` 这样的自然文本格式
4. 中间格式应该是每个片段一个markdown的小说
5. 技术美术手动理解情节、强调重点，然后合理进行转换

### 设计方向调整

**原设计**：语义化标记系统（`[内心]`、`[说]`、`[停顿:中]`等）
- ❌ 编剧仍需要理解标记的含义
- ❌ 编剧仍需要关心节奏和强调

**新设计**：完全自然的小说格式
- ✅ 编剧只需要像写小说一样创作
- ✅ 使用最基础的Markdown格式（`*斜体*`、`**粗体**`）
- ✅ 技术美术手动理解情节和创作意图，然后转换

## 二、新设计方案

### 中间格式设计

#### 格式规范

1. **内心独白/旁白**：使用 `*斜体*` 标记
   - 这是主角的内心想法、回忆、分析、环境描写等
   - 例如：`*早晨的阳光透过梧桐叶片的缝隙，像某种精致的花边装饰。*`

2. **强调**：使用 `**粗体**` 标记
   - 用于关键念头、重要信息、时间等需要强调的内容
   - 例如：`*我看着她的背影，想起了一个问题。**时间不对。**哪里不对？*`

3. **对话**：使用自然格式 `角色名: "对话内容"`
   - 主角的对话不需要角色名，直接写对话内容
   - 其他角色的对话需要标注角色名（如 `B: "..."`、`C: "..."`）
   - 例如：
     ```
     她指着街角那家新开的包子店，眼睛亮晶晶的。"我们去试试那个吧？听说皮薄馅大，很香的。"
     
     B: "你刚才有没有看到？我们班那个H，刚才在走廊里……"
     ```

4. **换行**：自然换行即可，不需要特殊标记
   - 换行表示节奏停顿，技术美术会根据情节判断如何转换

#### 格式特点

- **完全自然**：编剧可以像写小说一样创作
- **最小标记**：只使用最基础的Markdown格式（`*斜体*`、`**粗体**`）
- **无技术细节**：编剧不需要理解任何技术标签或程序结构

### 转换方式调整

#### 原设计
- 语义化标记系统，有明确的转换规则
- 技术美术可以系统化转换

#### 新设计
- **手动理解**：技术美术需要理解文风原则和创作意图
- **艺术性转换**：根据情节判断节奏、强调、交互等
- **不是自动化**：这是理解创作意图和艺术性转换，而不是机械转换

## 五、转换示例

### 编剧格式

```markdown
---
片段ID: P1-DTL-T0805-Breakfast
时间: 08:05
阶段: P1
解锁条件: FLAG_Game_Start
描述: 阶段1 - 早餐
作用: 建立"虚构恋爱史"的开端；情感绑定。
---

*早晨的阳光透过梧桐叶片的缝隙，像某种精致的花边装饰。*

*这是新学校的第一周。一切都应该是完美的。*

她指着街角那家新开的包子店，眼睛亮晶晶的。"我们去试试那个吧？听说皮薄馅大，很香的。"

我看着她，心里涌起一种奇怪的情绪。不是爱——那种东西太虚假了。

是一种……占有欲？不对。是保护欲？也不对。

"你的胃不好，早上吃这么油腻的东西会不舒服的。"

B: "她眨了眨眼，像是被什么小东西绊了一下。"可是……偶尔试试也没关系吧？"

"偶尔？"我笑了笑，去旁边的便利店买了三明治。

"你看，这个蛋白含量高，营养更均衡。"

她接过三明治，咬了一口，然后露出一个标准化的笑容。"嗯，很好吃。"

*那种笑容。*

*我见过。在很久很久以前，另一个女孩也露出过这样的笑容。*

*那种被训练出来的、迎合式的笑容。*

"中午的时候，我们还是去那个老地方吧。"她走在前面，声音里没有任何起伏。

"还是老地方。"我重复着，仿佛在确认什么。

*我看着她的背影，想起了一个问题。*

***她今天露出的疲惫笑容，是从哪里学来的？***
```

### 技术美术转换后的程序格式

```typescript

export const P1_DTL_T0805_Breakfast: ScriptSegment = {
  id: 'P1-DTL-T0805-Breakfast',
  time: '08:05',
  description: '阶段1 - 早餐',
  loop: 'P1',
  unlockFlags: ['FLAG_Game_Start'],
  lines: [
    {
      type: 'narration',
      text: '早晨的阳光透过梧桐叶片的缝隙，像某种精致的花边装饰。{br}{delay:0.8}这是新学校的第一周。一切都应该是完美的。'
    },
    {
      type: 'dialogue',
      text: '她指着街角那家新开的包子店，眼睛亮晶晶的。"我们去试试那个吧？听说皮薄馅大，很香的。"'
    },
    {
      type: 'dialogue',
      text: '我看着她，心里涌起一种奇怪的情绪。不是爱——那种东西太虚假了。{br}{delay:0.8}是一种……占有欲？不对。是保护欲？也不对。{br}{delay:0.8}"你的胃不好，早上吃这么油腻的东西会不舒服的。"'
    },
    {
      type: 'dialogue',
      character: 'B',
      text: '她眨了眨眼，像是被什么小东西绊了一下。"可是……偶尔试试也没关系吧？"'
    },
    {
      type: 'dialogue',
      text: '"偶尔？"我笑了笑，去旁边的便利店买了三明治。{br}{delay:0.8}"你看，这个蛋白含量高，营养更均衡。"{br}{delay:0.8}她接过三明治，咬了一口，然后露出一个标准化的笑容。"嗯，很好吃。"'
    },
    {
      type: 'narration',
      text: '那种笑容。{br}{delay:0.8}我见过。在很久很久以前，另一个女孩也露出过这样的笑容。{br}{delay:0.8}那种被训练出来的、迎合式的笑容。'
    },
    {
      type: 'dialogue',
      text: '"中午的时候，我们还是去那个老地方吧。"她走在前面，声音里没有任何起伏。{br}{delay:0.8}"还是老地方。"我重复着，仿佛在确认什么。'
    },
    {
      type: 'narration',
      text: '我看着她的背影，想起了一个问题。{br}{delay:1.0}{bold}她今天露出的疲惫笑容，是从哪里学来的？{/bold}'
    }
  ]
}
```

### 转换要点

1. **文本类型判断**：
   - `*斜体*` → `narration`
   - 直接对话（无角色名）→ `dialogue`（无character）
   - `B: "..."` → `dialogue`（有character）

2. **节奏与停顿处理**：
   - 根据情节判断，在关键情绪节点使用 `{br}{delay:0.8}` 或 `{br}{delay:1.0}`
   - 合并同一思绪流的内容，避免过度拆分

3. **视觉强调处理**：
   - `**粗体**` → `{bold}...{/bold}`
   - 如果涉及时间信息，必须使用 `{red}{typewriter}HH:MM{/typewriter}{/red}` 组合
