## 中间格式设计

### 设计原则

1. **自然性**：编剧可以像写小说一样创作
2. **语义化**：使用语义标记而非技术标记
3. **可转换性**：技术美术可以系统化转换
4. **可读性**：人类可读，便于审查和修改

### 核心标记

#### 文本类型
- `[内心]` - 主角内心独白（narration）
- `[说]` - 主角说出口的话（dialogue，无character）
- `[角色名]` - 其他角色对话（dialogue，有character）

#### 节奏与停顿
- `[停顿:短]` - 自然停顿（delay:0.5）
- `[停顿:中]` - 句间呼吸（delay:0.8）
- `[停顿:长]` - 段落转折（delay:1.0）

#### 视觉强调
- `[时间:HH:MM]` - 时间信息（red + typewriter）
- `[强调]` - 关键念头（bold）
- `[危险]` - 危险/执念（red）
- `[模糊]...[模糊]` - 记忆模糊（blur）
- `[修正:旧] [强调:新]` - 记忆修正（strike + red）
- `[逐字]` - 逐字强调（typewriter）

#### 交互元素
- `[选择]` - 选择分支（choice）
- `[标记:FLAG]` - 设置标记（command: setFlag）
- `[输入时间]` - 时间输入（input）
- `[时间选择]` - 时间匹配分支（timeChoice）

### 转换映射

| 中间格式 | 程序格式 |
|---------|---------|
| `[内心]` | `{ type: 'narration' }` |
| `[说]` | `{ type: 'dialogue' }` |
| `[角色名]` | `{ type: 'dialogue', character: '角色名' }` |
| `[停顿:短]` | `{delay:0.5}` |
| `[停顿:中]` | `{delay:0.8}` |
| `[停顿:长]` | `{delay:1.0}` |
| `[时间:HH:MM]` | `{red}{typewriter}HH:MM{/typewriter}{/red}` |
| `[强调]` | `{bold}...{/bold}` |
| `[危险]` | `{red}...{/red}` |
| `[模糊]...[模糊]` | `{blur}...{/blur}` |
| `[修正:旧] [强调:新]` | `{strike}旧{/strike} {red}新{/red}` |
| `[逐字]` | `{typewriter}...{/typewriter}` |
| `[选择]` | `{ type: 'choice', choices: [...] }` |
| `[标记:FLAG]` | `{ type: 'command', command: 'setFlag', params: { flag: 'FLAG' } }` |

### 问题诊断

**系统性根因**：文档设计时缺乏**角色分离**和**格式抽象层**的设计思维

### 解决方案

1. **职责分离**：
   - 编剧专注创作，使用语义化标记
   - 技术美术专注转换，使用程序格式
   - 中间格式作为桥梁

2. **格式抽象层**：
   - 创建中间格式，让编剧无需理解技术细节
   - 建立转换规则，让技术美术系统化转换
   - 保持格式的可扩展性

3. **文档拆分**：
   - 编剧文档：创作指导，不包含技术细节
   - 技术美术文档：转换规则，不包含创作指导
