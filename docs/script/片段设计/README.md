## 使用说明

### 对于结构设计者

1. 根据片段设计数据，设计该片段的结构大纲
2. 明确场景、人物、事件、情感弧和关键信息点
3. 参考 `knowledge_base.md` 中的：
   - 人物特征清单
   - 叙事诡计执行清单
   - 文风检查清单

### 对于初稿撰写者

1. 基于结构大纲，撰写具体的剧情文本
2. 遵循 `中间格式规范.md` 中的格式要求
3. 确保符合 `knowledge_base.md` 中的文风要求

## 片段设计字段说明

每个片段包含以下字段：

- **ID**: 片段的唯一标识符（格式：P阶段-类型-时间-名称）
- **作用**: 片段在游戏中的作用
- **进入时间**: 玩家输入的时间（HH:MM格式）
- **进入条件**: 解锁该片段所需的Flag或关键词
- **内容概述**: 片段的简要描述
- **诡计执行**: 该片段执行的叙事诡计
- **给予的线索**: 该片段给予玩家的线索（时间点、关键词等）
- **设置的Flag**: 该片段设置的Flag
- **伏笔**: 该片段埋下的伏笔

---

## 注意事项

1. **不要直接修改这些文件**：这些是设计数据，不是最终文本
2. **保持字段完整性**：每个片段都应该包含所有必需字段
3. **关注诡计执行**：每个片段都应该明确服务于某个叙事诡计
4. **信息密度**：每个片段都应该有高密度信息（线索、伏笔、人物特征等）
