# 中间格式规范

> **目标读者**：编剧、技术美术  
> **使用场景**：编剧使用此格式创作，技术美术使用此格式转换  
> **核心原则**：编剧完全专注于创作，像写小说一样自然写作。技术美术理解情节和创作意图，手动转换为程序格式。

---

## 概述

中间格式是一种**完全自然的小说格式**，编剧只需要像写小说一样创作，不需要关心任何技术细节。

---

## 格式结构

### 一、片段元数据（YAML Front Matter）

在文件开头使用 YAML Front Matter 格式标记片段信息：

```markdown
---
片段ID: P1-DTL-T0805-Breakfast
时间: 08:05
阶段: P1
解锁条件: FLAG_Game_Start
描述: 阶段1 - 早餐
作用: 建立"虚构恋爱史"的开端；情感绑定。
---
```

**字段说明**：
- `片段ID`：片段的唯一标识符（格式：P阶段-类型-时间-名称）
- `时间`：片段对应的时间点（HH:MM格式）
- `阶段`：片段所属的游戏阶段（P0-P6）
- `解锁条件`：解锁该片段所需的标记（多个标记用逗号分隔）
- `描述`：片段的简短描述（用于开发调试）
- `作用`：片段在游戏中的作用（用于理解创作意图）

### 二、文本内容

编剧使用自然格式创作，只需要：

1. **内心独白/旁白**：使用 `*斜体*` 标记
2. **强调**：使用 `**粗体**` 标记
3. **对话**：使用自然格式 `角色名: "对话内容"`
4. **换行**：自然换行即可

## 格式规范

### 一、内心独白/旁白

使用 `*斜体*` 标记，表示主角的内心想法、回忆、分析、环境描写等。

**格式**：
```markdown
*文本内容*
```

**示例**：
```markdown
*早晨的阳光透过梧桐叶片的缝隙，像某种精致的花边装饰。*

*这是新学校的第一周。一切都应该是完美的。*
```

**说明**：
- 这是主角的内心想法、回忆、分析、环境描写等
- 对应程序格式的 `narration` 类型
- 技术美术会根据情节判断如何转换

### 二、强调

使用 `**粗体**` 标记，表示关键念头、重要信息、时间等需要强调的内容。

**格式**：
```markdown
**文本内容**
```

**示例**：
```markdown
*我看着她的背影，想起了一个问题。*

***她今天露出的疲惫笑容，是从哪里学来的？***
```

**说明**：
- 用于关键念头、重要信息、时间等需要强调的内容
- 技术美术会根据情节判断如何转换（bold、red、typewriter等）

### 三、对话

使用自然格式，主角的对话不需要角色名，其他角色的对话需要标注角色名。

**格式**：
```markdown
"对话内容"  // 主角的对话

角色名: "对话内容"  // 其他角色的对话
```

**示例**：
```markdown
她指着街角那家新开的包子店，眼睛亮晶晶的。"我们去试试那个吧？听说皮薄馅大，很香的。"

B: "她眨了眨眼，像是被什么小东西绊了一下。"可是……偶尔试试也没关系吧？"
```

**说明**：
- 主角的对话直接写对话内容，不需要角色名
- 其他角色的对话需要标注角色名（如 `B: "..."`、`C: "..."`）
- 对应程序格式的 `dialogue` 类型
- 技术美术会根据情节判断如何转换

### 四、换行

自然换行即可，不需要特殊标记。

**说明**：
- 换行表示节奏停顿，技术美术会根据情节判断如何转换
- 可能需要 `{br}`、`{delay}` 等技术标签，但这是技术美术的工作

## 完整示例

```markdown
---
片段ID: P1-DTL-T0805-Breakfast
时间: 08:05
阶段: P1
解锁条件: FLAG_Game_Start
描述: 阶段1 - 早餐
作用: 建立"虚构恋爱史"的开端；情感绑定。
---

*早晨的阳光透过梧桐叶片的缝隙，像某种精致的花边装饰。*

*这是新学校的第一周。一切都应该是完美的。*

她指着街角那家新开的包子店，眼睛亮晶晶的。"我们去试试那个吧？听说皮薄馅大，很香的。"

我看着她，心里涌起一种奇怪的情绪。不是爱——那种东西太虚假了。

是一种……占有欲？不对。是保护欲？也不对。

"你的胃不好，早上吃这么油腻的东西会不舒服的。"

B: "她眨了眨眼，像是被什么小东西绊了一下。"可是……偶尔试试也没关系吧？"

"偶尔？"我笑了笑，去旁边的便利店买了三明治。

"你看，这个蛋白含量高，营养更均衡。"

她接过三明治，咬了一口，然后露出一个标准化的笑容。"嗯，很好吃。"

*那种笑容。*

*我见过。在很久很久以前，另一个女孩也露出过这样的笑容。*

*那种被训练出来的、迎合式的笑容。*

"中午的时候，我们还是去那个老地方吧。"她走在前面，声音里没有任何起伏。

"还是老地方。"我重复着，仿佛在确认什么。

*我看着她的背影，想起了一个问题。*

***她今天露出的疲惫笑容，是从哪里学来的？***
```

## 转换说明

### 编剧视角

**你只需要**：
- ✅ 像写小说一样自然创作
- ✅ 使用 `*斜体*` 表示内心独白/旁白
- ✅ 使用 `**粗体**` 表示强调
- ✅ 使用自然对话格式
- ✅ 自然换行

**你不需要**：
- ❌ 理解技术标签（`{br}`, `{delay}`等）
- ❌ 理解程序结构（narration vs dialogue）
- ❌ 关心节奏控制（技术美术会根据情节判断）

### 技术美术视角

**你需要**：
- ✅ 理解文风原则和创作意图
- ✅ 根据情节判断文本类型（narration vs dialogue）
- ✅ 根据情节判断节奏停顿的位置和时长
- ✅ 根据情节判断视觉强调的方式
- ✅ 手动转换为程序格式

**转换原则**：
- 理解创作意图，而不是机械转换
- 根据情节判断，而不是规则套用
- 艺术性转换，而不是自动化处理

---

## 注意事项

1. **文本类型判断**：正确区分 `*斜体*`（narration）和直接对话（dialogue）
2. **强调处理**：使用 `**粗体**` 表示强调，技术美术会根据情节判断如何转换
3. **换行处理**：自然换行即可，技术美术会根据情节判断如何转换
4. **时间信息**：在文本中自然提及时间，技术美术会根据情节判断是否需要特殊处理
